# GitHub Workflow Strategy

## Repository Recommendation
**Name:** `track1-restaurant`
**URL:** `https://github.com/mramzan579/Resturant.git`

## Repository Purpose
To store the frozen version of the premium restaurant sales funnel website, acting as a reliable portfolio asset for deployment and a template for future client customizations.

## Versioning Strategy
- **`main` branch:** The stable, production-ready, frozen version of the project used for live portfolio deployment (Netlify/Vercel).
- **Client branches (future):** When onboarding a new client based on this template, create a dedicated branch (e.g., `client/bistro-name`) to handle their custom logo, colors, and menu data without altering the master portfolio template.

## Commit Strategy
Commit messages should be clear, action-oriented, and indicate what part of the funnel was built or modified.

### Example Commits History
- `Initial Restaurant Funnel structure and basic layout`
- `Added sticky header, hero section, and typography styling`
- `Implemented JavaScript digital menu filtering and tabs`
- `Added WhatsApp ordering funnel and live experience timeline`
- `Conversion Features Added: Exit intent popup, live occupancy widget`
- `Final Outreach Version: Project frozen for portfolio`
- `Portfolio Assets Added: Summary, selling points, and deployment zip`
